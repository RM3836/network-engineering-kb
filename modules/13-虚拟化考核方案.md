# 13 - 虚拟化技术课程考核方案

## 知识点总结

### 考核目标
- 需求分析与架构设计能力
- 技术应用与规划能力
- 综合应用与方案设计能力
- 文档编写与专业表达能力

### 报告核心结构（5部分，约5000字）

```
1. 项目需求分析
   ├── 企业背景（自拟中小型企业）
   ├── 业务驱动需求（服务器整合、业务连续性、简化运维）
   └── 技术指标（VM数量、SLA、性能、预留扩展）

2. 整体架构设计
   ├── 逻辑拓扑图（管理集群、业务集群、网络分区、存储）
   ├── 硬件选型（ESXi主机数量、CPU/内存/网卡）
   ├── 软件许可（vSphere版本 + 功能需求）
   └── 核心组件（vCenter部署模式、ESXi部署方式）

3. 详细设计模块
   ├── 计算资源（集群规划、HA/DRS策略、亲和性规则）
   ├── 存储资源（FC/iSCSI/NFS/vSAN选型、数据存储分层）
   ├── 网络资源（vSS/vDS选型、端口组、VLAN隔离）
   ├── 虚拟机设计（模板、黄金镜像、资源配置建议）
   └── 运维管理（RBAC权限、监控、升级策略）

4. 预算与实施计划
   ├── 软件许可成本
   ├── 硬件成本
   └── 分阶段实施（试点→迁移→上线）

5. 总结与体会
```

### 关键设计要素

#### 计算资源设计

| 配置项 | 说明 |
|--------|------|
| 集群划分 | 管理集群 + 生产集群 + 测试集群 |
| HA策略 | 主机故障自动重启VM |
| DRS策略 | 自动化级别、迁移阈值 |
| 亲和性规则 | 同类VM聚合 或 关键VM分散 |
| 主机配置文件 | 标准化ESXi配置 |

#### 存储资源设计

| 类型 | 适用场景 |
|------|----------|
| FC | 高性能核心数据库（低延迟、高IOPS） |
| iSCSI | 中等性能应用（基于以太网、成本低） |
| NFS | 文件共享、非关键业务 |
| vSAN | 超融合架构（利用本地硬盘组建存储池） |

#### 网络资源设计

| 网络类型 | VLAN | 用途 |
|----------|------|------|
| 管理网络 | VLAN 10 | ESXi/vCenter管理 |
| vMotion网络 | VLAN 20 | 在线迁移 |
| 存储网络 | VLAN 30 | FC/iSCSI存储 |
| FT网络 | VLAN 40 | Fault Tolerance |
| 业务网络 | VLAN 50+ | 按部门/应用划分 |

#### 虚拟机模板

| VM类型 | vCPU | 内存 | 存储 | 存储类型 |
|--------|------|------|------|----------|
| Web服务器 | 2 | 4GB | 50GB | SAS |
| 数据库服务器 | 4 | 16GB | 200GB | SSD |
| 文件服务器 | 2 | 8GB | 500GB | SAS |
| 开发测试VM | 2 | 4GB | 100GB | SAS |

### RBAC权限设计

```
vCenter
├── 管理员角色: 完全控制
├── VM管理员: 创建/删除/修改VM
├── 存储管理员: 管理数据存储
├── 网络管理员: 管理vDS/端口组
└── 只读审计员: 仅查看
```

## 实操步骤

### 方案设计流程

```
1. 需求分析
   → 确定企业背景和业务需求
   → 量化技术指标（VM数、SLA、性能）

2. 架构设计
   → 画逻辑拓扑图（draw.io/Lucidchart）
   → 选择vSphere版本和许可

3. 详细设计
   → 计算: 集群规划 + HA/DRS配置
   → 存储: FC/iSCSI选型 + 分层策略
   → 网络: vDS + VLAN规划
   → VM: 模板 + 资源配置

4. 预算估算
   → 软件: vSphere许可费（按CPU数）
   → 硬件: 服务器 + 交换机 + 存储
   → 其他: 培训 + 运维

5. 实施计划
   → 试点阶段（1-2台ESXi）
   → 迁移阶段（P2V物理机转虚拟机）
   → 全面上线
```

## 常见问题与排错

| 问题 | 原因 | 解决 |
|------|------|------|
| 方案不切实际 | 需求分析不充分 | 先调研再设计 |
| 架构图不清晰 | 未分层展示 | 按物理/虚拟/网络/存储分层 |
| 预算超支 | 功能过度配置 | 按需选型，预留20%扩展 |
| 缺少冗余设计 | 忽略单点故障 | 关键组件双冗余 |

## 速记口诀

> **需求分析量化指标，分层架构画拓扑图，计算存储网络三设计，RBAC权限分角色管。**
