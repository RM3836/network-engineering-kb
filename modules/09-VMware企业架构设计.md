# 09 - VMware vSphere 8 企业级架构设计

## 知识点总结

### 架构分层
```
管理与监控层 (vCenter + Update Manager + SNMP)
    │
虚拟层 (ESXi + 资源池 + HA/DRS)
    │
网络层 (vDS分布式交换机 + VLAN隔离)
    │
存储层 (FC存储阵列 + SSD/SAS分层)
    │
物理层 (服务器集群 + 交换机 + 存储)
```

### 核心设计原则
- **分层解耦 + 集中管理**: 各层职责清晰，vCenter统一管控
- **冗余设计**: 双电源、双HBA卡、双控制器、双交换机
- **性能分层**: SSD高性能存储 + SAS大容量存储

### 硬件规划

| 集群 | 数量 | 单台配置 | 用途 |
|------|------|----------|------|
| 管理生产集群 | 4台 | 2×Xeon Gold 6330(48核), 128GB, 2×1.2TB SAS RAID1, 4×10Gb网卡, 2×16Gb FC HBA | 核心业务+管理VM |
| 开发测试集群 | 2台 | 同上 | 测试+非核心VM |

### 存储分层设计

| 存储类型 | 介质 | 容量 | IOPS | 用途 |
|----------|------|------|------|------|
| SSD数据存储 | SSD | 80TB | ≥15000 | 数据库VM、关键应用 |
| SAS数据存储 | SAS | 120TB | ≥5000 | Web服务器、文件服务器、测试VM |

**存储策略**:
- SSD: 厚置备置零 + SIOC高优先级 + Round Robin多路径
- SAS: 厚置备延迟置零 + SIOC中优先级 + Round Robin多路径

### 网络设计

```
vDS分布式交换机
├── 管理网络 (VLAN 10)
├── vMotion网络 (VLAN 20)
├── 存储网络 (VLAN 30) - FC/iSCSI
├── FT网络 (VLAN 40) - Fault Tolerance
└── 业务网络 (VLAN 50+) - 按部门划分
```

### 高可用设计

| 组件 | 冗余方式 |
|------|----------|
| 服务器电源 | 双电源，接入不同供电回路 |
| 网卡 | 4×10Gb，LACP链路聚合 |
| 存储连接 | 双HBA卡 → 双控制器(A/B) |
| 交换机 | 双核心交换机，冗余链路 |
| NTP | 双NTP服务器 |

### 虚拟机模板管理
- **黄金镜像**: 标准化VM模板，预装OS+基础软件
- **自定义规范**: 自动化IP、主机名、域名配置
- **资源配置建议**:
  - Web服务器: 2vCPU, 4GB, 50GB
  - 数据库服务器: 4vCPU, 16GB, 200GB SSD
  - 文件服务器: 2vCPU, 8GB, 500GB SAS

### 权限与安全 (RBAC)

| 角色 | 权限 |
|------|------|
| 虚拟机管理员 | 创建/删除/修改VM |
| 只读审计员 | 只能查看，不能操作 |
| 存储管理员 | 管理数据存储和LUN |

## 实操步骤

### 1. 部署ESXi主机
```bash
# 制作U盘启动盘(Rufus: GPT+UEFI)
# 裸机安装 → 设置root密码 → 配置管理网络IP
# 浏览器访问 https://<ESXi-IP>:9443 验证
```

### 2. 部署vCenter (vCSA)
```bash
# 阶段1: 挂载ISO → installer.exe → 部署到ESXi
# 阶段2: 配置NTP、SSO域名(vsphere.local)
# 浏览器访问 https://<vCenter-IP>:443 验证
```

### 3. 创建集群与配置HA/DRS
```
vCenter → 创建数据中心 → 创建集群
→ 启用HA(主机故障自动重启VM)
→ 启用DRS(负载均衡自动调度)
→ 添加ESXi主机到集群
→ 配置存储(添加LUN)
→ 配置网络(vDS + 端口组)
```

### 4. 存储多路径配置
```
ESXi → 存储 → 存储设备 → 选择LUN
→ 编辑多路径 → Round Robin(轮询)
```

## 常见问题与排错

| 问题 | 原因 | 解决 |
|------|------|------|
| vMotion失败 | 网络不通/CPU不兼容 | 检查vMotion端口组 + EVC模式 |
| HA隔离 | 管理网络断开 | 检查心跳网络配置 |
| 存储性能低 | 单路径/未启用SIOC | 启用多路径+SIOC |
| VM启动失败 | 资源不足 | 检查集群资源预留 |

## 速记口诀

> **四层架构分层管，双冗余防单点故障，SSD高性能SAS大容量，HA保可用DRS均负载。**
