# 01 - 环境准备与虚拟机配置

## 知识点总结

### 核心概念
- **双网卡架构**: NAT模式（上网用）+ 仅主机模式（内网通信）
- **CentOS Stream 9**: RHEL上游滚动发行版，适合学习企业级Linux
- **Docker基础**: 容器化部署的前提，所有后续模块的基石

### 网络规划

| 主机名 | 仅主机IP | 角色 |
|--------|----------|------|
| server-node | 192.168.50.10 | Ansible控制端 + Zabbix服务器 |
| client-web | 192.168.50.20 | Web服务器（Nginx+Tomcat） |
| client-db | 192.168.50.30 | 数据库服务器（MySQL） |

## 实操步骤

### 1. 关闭防火墙和SELinux

```bash
# 关闭防火墙
sudo systemctl stop firewalld
sudo systemctl disable firewalld

# 关闭SELinux（立即生效 + 永久生效）
sudo setenforce 0
sudo sed -i 's/SELINUX=enforcing/SELINUX=disabled/g' /etc/selinux/config
```

### 2. 配置仅主机模式网卡

```bash
# 服务器 (192.168.50.10)
sudo nmcli connection modify ens224 ipv4.addresses 192.168.50.10/24 ipv4.method manual
sudo nmcli connection up ens224

# 客户机1 (192.168.50.20)
sudo nmcli connection modify ens224 ipv4.addresses 192.168.50.20/24 ipv4.method manual
sudo nmcli connection up ens224

# 客户机2 (192.168.50.30)
sudo nmcli connection modify ens224 ipv4.addresses 192.168.50.30/24 ipv4.method manual
sudo nmcli connection up ens224
```

### 3. 配置主机名和hosts

```bash
# 设置主机名（分别在对应机器执行）
sudo hostnamectl set-hostname server-node
sudo hostnamectl set-hostname client-web
sudo hostnamectl set-hostname client-db

# 配置hosts（所有机器执行）
echo "192.168.50.10 server-node" | sudo tee -a /etc/hosts
echo "192.168.50.20 client-web" | sudo tee -a /etc/hosts
echo "192.168.50.30 client-db" | sudo tee -a /etc/hosts
```

### 4. 安装Docker

```bash
# 安装Docker
sudo dnf install -y dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 验证
sudo systemctl status docker
docker --version
```

### 5. 验证互通性

```bash
# 从服务器ping两台客户机
ping -c 3 192.168.50.20
ping -c 3 192.168.50.30
```

## 常见问题与排错

| 问题 | 原因 | 解决 |
|------|------|------|
| 网卡名不是ens224 | 虚拟机网卡命名不同 | `nmcli device` 查看实际网卡名 |
| ping不通 | 仅主机网卡未激活 | `nmcli connection up ens224` |
| Docker安装失败 | 源不可用 | 换用阿里云镜像源 |
| SELinux关闭后仍拦截 | 需重启生效 | `reboot` 重启 |

## 速记口诀

> **关防关SELinux，双网卡配静态IP，主机名hosts同步装，Docker自启ping验证。**
